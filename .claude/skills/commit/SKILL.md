コミット対象に機密情報・個人情報が含まれていないか検査してからコミットする。

## 手順

### 1. コミット対象を確認
- `git status` と `git diff HEAD` で、これからコミットされる内容（追跡中の変更＋ステージ済み）を把握する。
- 新規ファイルは `git diff HEAD -- <file>` などで中身も確認する。

### 2. 漏洩検査
追加・変更された差分の中に以下が含まれていないか確認する。
- **機密情報**: APIキー、トークン、シークレット、パスワード、`service_role` キー、接続文字列、`.env*` の値
- **クレジットカード情報**: カード番号、CVV、有効期限
- **個人情報**: 氏名、住所、電話番号、メールアドレス（コミット作者の git 設定は対象外）

判断に迷うものは「疑わしい」として NG 扱いにする。

### 3. 判定
- **OK（漏洩なし）**: そのままコミットする。
  - コミットメッセージは変更内容に即した日本語の Conventional Commits 形式（既存履歴に合わせる）。
  - push はしない。
- **NG（漏洩の疑いあり）**: コミットせず、何が・どのファイルのどこにあるかを報告する。対処法を簡潔に添える。

## ルール
- push は行わない。
- NG の時は絶対にコミットしない。
- 検査で見つけた機密情報の実値は報告に書かない（ファイル名と種別のみ）。
